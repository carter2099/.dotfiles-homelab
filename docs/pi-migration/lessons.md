## Pre-migration observations

- Pi already installed and authenticated before migration planning began — interactive use was the canary
- `@juicesharp/rpiv-web-tools` was pre-installed to close the web-search gap
- The OpenCode Go subscription models map 1:1 to Pi's `opencode-go` provider — no model loss
- Pi's `-p` (print) mode is the direct equivalent of `opencode run` for headless/automated use
